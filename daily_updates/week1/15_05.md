### Legacy Library Embed in LateNightQML — Event-Driven Rendering & Keyboard Support

**Branch:** [`PoC-Legacy-Library-QML-Integration`](https://github.com/xARSENICx/mixxx/tree/PoC-Legacy-Library-QML-Integration)

- **Event-Driven Rendering (Performance Optimization):**
  Replaced the fixed 33ms repaint timer with an event-driven `requestRender()` scheduler. Repaints now only happen when the embedded `QWidget` tree actually changes. This prevents blocking the main thread and allows us to accurately measure the performance impact of QML waveforms and animations. Also fixed a repaint feedback loop.
  _Commits: `402862f`, `d318382`_

- **Keyboard Forwarding & Search Resolution:**
  Implemented `keyPressEvent` and `keyReleaseEvent` overrides to forward QML keyboard events to the embedded widget tree. Synthesized `FocusIn/FocusOut` events and implemented focus tracking so the `WSearchLineEdit` can accept text and show its cursor. Restored the missing `Library::search` connection to actually filter the track list, and prepended `default.qss` to fix missing search/clear icons.
  _Commit: `eac3ad8`_

- **Visual Stability:**
  Hardcoded a dark `QPalette` for the offscreen `QWidget` tree to prevent native OS light-mode themes from bleeding into elements like scrollbars, fixing visual inconsistencies.
  _Commit: `f8c4d35`_

- **Skin Updates:**
  Updated `skin_preview.png` for LateNightQML to reflect the current state of the UI (added for self-motivation as the skin becomes a reality).
  _Commit: `ecc172a`_

Investigated and submitted a fix for a QML waveform rendering artifact where the play-position marker (cursor) could visually disconnect after resizing an empty waveform display. The fix ensures QML marker nodes are properly marked dirty for the scene graph renderer and fixes the broken waveform cursor in LateNightQML.
